# Contributing Guide

このリポジトリへのコントリビュート規約です。  
従わなくても全然いいですが迷ったら参考にしてください。

## 1. ブランチ運用

- メイン開発ブランチ: `develop`
- リリース対象ブランチ: `main`
- 作業ブランチは `develop` から作成する

### ブランチ命名規則

- 機能追加: `feature/<issue番号>-<要約>`
- バグ修正: `fix/<issue番号>-<要約>`
- 雑務/保守: `chore/<要約>`
- ドキュメント: `docs/<要約>`

例:

- `feature/123-turn-timeout`
- `fix/245-invalid-action-order`
- `docs/contributing-update`

## 2. コミットメッセージ

Conventional Commits を推奨します。

- `feat: ターン終了処理を追加`
- `fix: 移動可能マス判定の不具合を修正`
- `docs: READMEに起動手順を追記`

1コミット1目的を基本にしてください。

## 3. Pull Request ルール

### PRタイトル

`<type>: <概要> (#issue番号)`

例:

- `feat: ターン終了タイマーを実装 (#123)`

### PR本文に必須で書くこと

- 変更の目的
- 変更内容の要点
- 動作確認手順
- 影響範囲（`game-client` / `game_server` / インフラなど）
- UI変更がある場合はスクリーンショット

### PRの粒度

- 1機能1PRを原則とする
- 大規模変更は段階的に分割する
- リファクタと仕様変更は可能な限り分ける

## 4. レビューとマージ

- 指摘がある場合は対応後に再レビュー依頼
- マージ方式は `Squash and merge` を推奨

## 5. コーディングとテスト

- 既存の設計・命名・レイヤ構成に合わせる
- 変更箇所に応じた最小限のテストを実施する
